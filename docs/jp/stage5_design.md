# ステージ5 —— 名詞同士の格関係と動詞の属性解析 (Case Relation & Attribute Analysis)
## 概要：
本ステップでは、動詞の語幹と活用（れる／られる）を論理的に分離し、格助詞との組み合わせによって「誰が誰に何をしたか（あるいはどういう状態か）」を確定させます。辞書形ノードを用いたネットワーク参照により、煩雑な活用形を「論理の核」へと還元し、曖昧さを解消します。

## 主要ポイント：
**1. 活用エンジンの正規化（Normalization）**：

「れる／られる」などの接辞を検知し、活用表テンプレートを参照して、常に「辞書形（語幹）」へ巻き戻します。これにより、辞書を肥大化させずにすべての活用形を論理ノードへ紐付けます。

**2. 動詞属性のネットワーク参照**：

辞書形ノードが保持する ⁠[+Stative]⁠（状態）や ⁠[+Dynamic]⁠（動作）フラグを参照します。この属性が、格解析における「矢印（因果）」と「等号（属性）」の決定的な判断基準となります。

**3. 格助詞による役割特定**：

「～に」という助詞の有無を検知し、動詞属性と照合することで、「受身（被受給者）」と「尊敬（行為者）」を論理的に分離します。

## 論理比較：格関係の解析
| 入力文 | 論理プロセス | AIの理解（ノード構成） |
|--|--|--|
| 「部長に褒められる。」 | [褒める] + [受身] + [に] | [部長] --(行為)--> [私(褒められる状態)] |
| 「部長が来られる。」 | [来る] + [尊敬] + [主語:部長] | [部長] == [尊大属性] (移動動作) |
| 「君に恋焦がれる。」 | [恋焦がれる] + [自発] + [に] | [私] == [心理状態: 恋焦がれ] |

## 格関係・属性追跡の例：
**入力**： 「彼女に好かれている。」

**解析**：
- 活用処理： 「好かれている」 → ⁠[好く]⁠ (辞書形) + ⁠[て+いる]⁠ (状態) + ⁠[れる]⁠ (受身)。
- 動詞属性： ⁠[好く]⁠ は心理動詞（状態）として定義。
- 格の特定： ⁠[彼女] + に⁠ が存在し、心理動詞の受身であるため、「心理的影響の流入」と判定。
- 事象チェーン： ⁠[彼女(原因/源)]⁠ → ⁠[好意という属性]⁠ → ⁠[私(状態: 好かれている)]⁠。

**AIの理解**：
外部（彼女）からの心理的な矢印が自分（私）へ向いている状態であることを正しく把握します。これは因果の「動作」ではなく、「属性の保持（等号）」としてログに圧縮されます。
